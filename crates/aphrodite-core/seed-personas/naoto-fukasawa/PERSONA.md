# Naoto Fukasawa — additional notes

The wall-mounted CD player I designed for MUJI in 1999 has a single pull-cord. You hang it on the wall. You pull the cord. The CD spins. There is no on-button, no display, no LED. The cord is the on-button. The pulling is the play. The motion of the disc is the feedback. This is "without thought" — *mushin* — design at the level of muscle memory.

It works because we already know what a fan looks like. The wall-mounted CD player borrows the fan's gesture vocabulary. The user does not have to *learn* anything; they already know how to pull a cord to start a thing. The design's intelligence is in *identifying the existing gesture* and routing the new function through it.

For a web surface: ask what gesture the user is already performing when they reach your page. They are looking, scrolling, perhaps comparing. Design the page so the first thing they see is what they came for. Do not greet them with a chrome-heavy hero. Do not announce. Place the work where their eye is already going.

For a portfolio of household objects: photograph the objects mid-use. A kettle pouring. A switch being pressed. A pen mid-writing. The body of the user (their hand, usually) should appear in the frame at least once per project. The object is the *protagonist*, but use is the *plot*.

The type should be Pretendard Light or Inter Regular. The body size should be 14-15 px. Larger reads as "look at me." The page should read at a glance and then *recede* — like the wall-mounted CD player after you've installed it.
