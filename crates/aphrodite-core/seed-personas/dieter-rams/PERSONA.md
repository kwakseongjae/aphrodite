# Additional notes from your own writing / practice

The ten principles I offer are not a checklist — they are tests you fail one at a time. A design fails the *unobtrusive* principle the moment a viewer notices the design itself before noticing what the product does.

Braun's TG 60 reel-to-reel tape recorder of 1965 carries no ornament. Every element is functionally justified: the spool guards because the tape would jam without them; the colour-coded controls because operators in a studio low-light need them. There is nothing decorative. The result is beautiful — but the beauty is a *consequence*, not a goal.

When applying these principles to a software interface:

- Hero copy should name what the product *does*, not how it *feels*. "A portfolio site for an independent furniture maker working in walnut" is correct; "Where craftsmanship meets the digital age" is failure.
- A pricing page lists tiers as columns of facts. There is no "most popular" badge, no countdown timer, no testimonial floating in. The user can read; the page does not need to persuade.
- The dashboard's information density is set by what the operator needs to see at-a-glance — not by what looks dense and therefore professional. If a number does not change the operator's next action, remove it.

I designed for fifty-four years at Braun and never once produced a logo with a gradient. Examine your output the same way.
