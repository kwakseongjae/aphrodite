# Pretendard — why it matters

Pretendard is the default to consider for any Korean-language web surface that needs Latin pairing. It was designed *bilingually* — the Hangul and Latin glyphs share x-height and stroke contrast, so a line mixing English and Korean reads as one typographic system instead of two pasted together. Most other Korean sans families (Nanum Gothic, Spoqa Han Sans Neo, Apple SD Gothic Neo) drift in x-height or weight when paired with Inter/Helvetica.

## What to absorb

- **Bilingual matched metrics.** When a brief mentions Seoul / Korean / 한국어 / Hangul, default to Pretendard *unless* the persona explicitly demands a serif Hangul (e.g. Ando → Nanum Myeongjo, Hara → Apple SD Gothic Neo Light).
- **Variable axis (wght 100–900).** Use the variable file when CSS variables can hold it; falls back gracefully to static weights.
- **MIT licensed.** Embed directly. No commercial-foundry friction.

## What NOT to copy

The Pretendard project site itself is a *typeface specimen* (centered, restrained, almost a Bauhaus poster). That layout is correct *for the typeface*. Do not copy the centered specimen layout for a portfolio or editorial surface — the typeface site is showing off the type; your surface uses the type to show off something else.

## Source signature

- Family: Pretendard
- Weights: 100, 200, 300, 400, 500, 600, 700, 800, 900 (variable)
- Pairing: any neutral Latin (Inter, Söhne, ABC Diatype)
- Optical sizing: no (use 14–17 px body, 36–60 px display)
