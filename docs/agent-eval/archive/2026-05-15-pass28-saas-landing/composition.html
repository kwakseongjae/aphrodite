Platform

Everything between your Next.js app and the user

EdgeFn handles routing, execution, and scaling so your team can focus on application logic — not infrastructure operations.

One-command deploys

Push API routes and middleware to the edge with a single CLI command. Auto-detects your Next.js project structure, bundles with esbuild, and deploys to every region in parallel.

40+ edge regions

Functions run in the region closest to each request — North America, Europe, Asia-Pacific, South America. Your users get sub-50ms responses regardless of origin.

Zero cold starts

V8 isolates keep your functions warm and resident. No container spin-up, no provisioning delay. First request executes as fast as the hundredth.

Preview deployments

Every pull request gets its own isolated environment with a unique URL. Test edge behavior in staging before merging — no shared environments, no surprises in production.

Built-in auth helpers

First-class middleware for session validation, JWT verification, and geofencing. Protect routes at the edge before they reach your origin — reduce load and improve security.

Real-time logs

Stream execution logs, request metadata, and performance metrics from every region. Filter by route, status code, or latency — debug edge issues in seconds, not hours.

How it works

From local to global in three steps

No infrastructure knowledge required. If you know Next.js, you already know EdgeFn.

01

Write your function

Standard Next.js API route or middleware. Use the Request and Response objects you already know — no proprietary wrappers, no SDK to learn.

02

Configure regions

Add an edgefn.config.ts to specify regions, environment variables, and routing rules — or accept the defaults and deploy everywhere instantly.

03

Deploy and monitor

Run npx edgefn deploy. Your functions go live across all regions in seconds. Open the dashboard to monitor latency, errors, and request volume in real time.

Developer experience

Configured in TypeScript, not YAML

Full type safety for your edge configuration. Route definitions, environment variables, and deployment settings — all in one typed config file.
